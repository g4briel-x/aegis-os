# Code of Conduct

## 1. Purpose

Aegis OS should remain a professional, respectful and constructive project environment.

---

## 2. Expected Behavior

Participants are expected to:

```text
communicate respectfully
focus on the work
give clear feedback
accept constructive review
document decisions
avoid personal attacks
respect different experience levels
```

---

## 3. Unacceptable Behavior

Unacceptable behavior includes:

```text
harassment
insults
threats
discrimination
public sharing of private information
deliberate disruption
malicious contributions
security abuse
```

---

## 4. Technical Review Culture

Reviews should focus on:

```text
correctness
clarity
maintainability
security
documentation quality
testability
repository consistency
```

---

## 5. Enforcement

Maintainers may:

```text
request changes
close issues
reject pull requests
remove harmful content
restrict participation
```

---

## 6. Final Principle

> A strong project culture protects both people and the quality of the system.
