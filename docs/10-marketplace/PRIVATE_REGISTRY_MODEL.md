1. Introduction

This document defines how private or organization-scoped Marketplace registries operate.

Private registries support internal distribution and controlled visibility.

2. Use Cases

Examples:

enterprise-only Skills;
regulated industry workflows;
internal security packages;
client-specific bundles.
3. Registry Controls

Private registries should support:

access control;
approval workflows;
scoped visibility;
audit logs;
internal trust policies.
4. Final Principle

Private registries enable controlled ecosystem growth without exposing sensitive intelligence assets.

Aegis OS — Marketplace Review and Rating Model

Version: 0.1
Status: Marketplace Document

1. Introduction

This document defines how Marketplace components can be reviewed and rated.

Reviews improve discoverability and quality feedback.

2. Review Goals

Reviews may help assess:

usefulness;
quality;
maintainability;
documentation clarity;
runtime reliability.
3. Rating Considerations

Possible dimensions:

quality;
trustworthiness;
usability;
support quality;
update discipline.
4. Review Safety

Review systems should protect against:

manipulation;
spam;
low-signal feedback;
unverifiable claims.
5. Final Principle

Reviews should improve ecosystem intelligence, not devolve into noise.
