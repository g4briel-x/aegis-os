# Aegis OS — Certification Model

Version: 0.1.0  
Status: Draft

## Purpose

This document defines how an Aegis OS asset becomes certified.

## Certification Requirements

An asset can be certified when it has:

```text
clear purpose
documented role
valid metadata
existing repository path
quality checklist
examples when useful
related assets when relevant
passing validation
review approval
```

## Certification Scope

Certification may apply to:

```text
skills
playbooks
patterns
templates
registries
scripts
CLI commands
documentation sections
```

## Certification Decision

Certification should answer:

```text
Is this asset accurate?
Is it complete enough?
Is it reusable?
Is it safe?
Is it consistent with Aegis OS standards?
```

## Final Principle

> Certification marks an asset as trusted enough for repeated use.
