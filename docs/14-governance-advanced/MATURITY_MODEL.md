# Aegis OS — Maturity Model

Version: 0.1.0  
Status: Draft

## Purpose

This document defines maturity levels for the Aegis OS project and its assets.

## Maturity Levels

```text
0 - idea
1 - draft
2 - structured
3 - usable
4 - validated
5 - operational
6 - certified
```

## Asset Maturity

An asset moves from draft to usable when:

```text
it has a clear role
it has a stable path
it appears in the registry if needed
it passes validation
it can be understood without hidden context
```

## System Maturity

Aegis OS moves toward maturity through:

```text
documentation completeness
registry reliability
CLI operability
test coverage
automation
runtime implementation
release discipline
```

## Final Principle

> Maturity should be proven by structure, validation and repeatable use.
