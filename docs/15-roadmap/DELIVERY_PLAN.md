# Aegis OS — Delivery Plan

Version: 0.1  
Status: Roadmap Document

---

# 1. Introduction

This document defines how roadmap items should be delivered over time.

---

# 2. Delivery Philosophy

Aegis OS follows this principle:

> Delivery should move from dependency-heavy foundations toward visible framework value.

---

# 3. Recommended Delivery Sequence

Recommended order:

1. documentation foundation;
2. core and shared stabilization;
3. Skills v2 and premium Skills;
4. Playbooks and Templates;
5. runtime and registry prototypes;
6. SDK and CLI surfaces;
7. Marketplace and ecosystem controls;
8. adoption assets and examples.

---

# 4. Delivery Constraints

Delivery should consider:

- architectural dependency order;
- contributor bandwidth;
- documentation debt;
- validation cost.

---

# 5. Final Principle

> Delivery order matters because some work unlocks everything that follows.
