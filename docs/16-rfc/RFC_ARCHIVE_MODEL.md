# Aegis OS — RFC Archive Model

Version: 0.1  
Status: RFC Document

---

# 1. Introduction

This document defines how historical RFCs are archived and preserved.

---

# 2. Archive Goals

The archive should preserve:

- rejected ideas;
- superseded designs;
- historical rationale;
- change evolution;
- long-term institutional memory.

---

# 3. Archive Rules

Archived RFCs should remain:

- readable;
- immutable except for status metadata;
- linked to superseding RFCs when relevant.

---

# 4. Final Principle

> RFC archives preserve design history so the project can learn from its own decisions.
