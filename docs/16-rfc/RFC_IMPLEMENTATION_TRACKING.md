# Aegis OS — RFC Implementation Tracking

Version: 0.1  
Status: RFC Document

---

# 1. Introduction

This document defines how accepted RFCs are tracked through implementation.

---

# 2. Tracking Goals

Implementation tracking should show:

- accepted design status;
- implementation start;
- milestone progress;
- deviations from the RFC;
- completion state.

---

# 3. Tracking Fields

Example:

```yaml
implementation_tracking:
  rfc_id:
  implementation_status:
  owner:
  milestones:
  deviations:
```

---

# 4. Final Principle

> An accepted RFC is not finished when approved; it is finished when implemented or intentionally superseded.
