# Aegis OS — RFC Priority Model

Version: 0.1  
Status: RFC Document

---

# 1. Introduction

This document defines how RFCs are prioritized.

---

# 2. Priority Factors

Priority may depend on:

- strategic importance;
- dependency unlocks;
- security urgency;
- ecosystem impact;
- implementation readiness.

---

# 3. Priority Levels

Suggested levels:

- P0 critical;
- P1 high;
- P2 medium;
- P3 exploratory.

---

# 4. Final Principle

> Priority should reflect ecosystem leverage, not only author enthusiasm.
