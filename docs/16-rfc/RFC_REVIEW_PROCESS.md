# Aegis OS — RFC Review Process

Version: 0.1  
Status: RFC Document

---

# 1. Introduction

This document defines how RFCs are reviewed.

RFC review exists to improve decision quality before major change is adopted.

---

# 2. Review Goals

The review process should evaluate:

- clarity of problem statement;
- design quality;
- trade-offs;
- architecture compatibility;
- security implications;
- migration impact.

---

# 3. Review Participants

Possible participants:

- component owners;
- domain owners;
- maintainers;
- security reviewers;
- architecture reviewers.

---

# 4. Review Flow

```text
RFC Submitted

   ↓

Initial Triage

   ↓

Domain Review

   ↓

Architecture / Security Review

   ↓

Feedback Consolidation

   ↓

Revision or Decision
```

---

# 5. Final Principle

> RFC review is most useful when it challenges assumptions early and constructively.
