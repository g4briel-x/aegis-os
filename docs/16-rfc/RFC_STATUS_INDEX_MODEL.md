# Aegis OS — RFC Status Index Model

Version: 0.1  
Status: RFC Document

---

# 1. Introduction

This document defines how RFC status should be indexed and tracked.

---

# 2. Index Goals

The RFC index should make visible:

- active RFCs;
- accepted RFCs;
- rejected RFCs;
- superseded RFCs;
- implementation status.

---

# 3. Example Index Fields

```yaml
rfc:
  id:
  title:
  status:
  category:
  owner:
  created:
  last_updated:
```

---

# 4. Final Principle

> A visible RFC index turns proposal history into navigable project memory.
