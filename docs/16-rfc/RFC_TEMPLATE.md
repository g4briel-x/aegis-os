# 1. Introduction

This document defines the standard structure for RFCs inside Aegis OS.

---

# 2. Standard RFC Structure

```markdown
# RFC Title

## Status

## Authors

## Created

## Summary

## Motivation

## Problem Statement

## Goals

## Non-Goals

## Proposed Design

## Alternatives Considered

## Trade-Offs

## Compatibility Impact

## Security Considerations

## Operational Considerations

## Migration Plan

## Open Questions

## Decision Required
```

---

# 3. Template Philosophy

The RFC template should force clarity around:

- motivation;
- scope;
- trade-offs;
- impact;
- open risk.

---

# 4. Final Principle

> A good RFC is specific enough to review and concrete enough to act on.
