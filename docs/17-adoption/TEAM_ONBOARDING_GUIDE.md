# Aegis OS — Team Onboarding Guide

Version: 0.1  
Status: Adoption Guide

---

# 1. Introduction

This guide defines how a team can start using Aegis OS collaboratively.

---

# 2. Team Onboarding Steps

Recommended sequence:

1. Assign a project owner.
2. Review foundation documents.
3. Define the team's use case.
4. Select initial Skills or Playbooks.
5. Establish contribution rules.
6. Define review process.
7. Create first internal examples.
8. Commit changes with clear version history.

---

# 3. Team Roles

Suggested roles:

- project owner;
- documentation maintainer;
- Skill author;
- reviewer;
- domain expert;
- operations owner.

---

# 4. First Team Workshop

Recommended agenda:

- explain Aegis OS vision;
- review repository structure;
- select first use case;
- map required Skills;
- define first deliverable.

---

# 5. Final Principle

> Team adoption works best when ownership, use case and contribution rules are clear from the beginning.
