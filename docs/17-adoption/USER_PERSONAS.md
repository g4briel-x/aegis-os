# Aegis OS — User Personas

Version: 0.1  
Status: Adoption Document

---

# 1. Introduction

This document defines key user personas for Aegis OS.

Personas help align documentation, examples and adoption flows with real user needs.

---

# 2. Solo Builder

Needs:

- simple structure;
- reusable expert prompts;
- clear documentation;
- fast production workflow.

Typical use:

- building personal AI workflows;
- creating expert Skills;
- organizing knowledge.

---

# 3. Developer

Needs:

- repository structure;
- CLI/SDK guidance;
- validation workflows;
- extension points.

Typical use:

- building components;
- automating workflows;
- integrating Aegis OS into tools.

---

# 4. Product Manager

Needs:

- product strategy Skills;
- discovery Playbooks;
- roadmap Templates;
- decision support.

Typical use:

- product planning;
- requirements definition;
- market analysis.

---

# 5. Technical Team

Needs:

- shared standards;
- multi-role Skills;
- review workflows;
- governance.

Typical use:

- engineering enablement;
- architecture support;
- operational playbooks.

---

# 6. Enterprise

Needs:

- governance;
- security;
- private registries;
- compliance;
- adoption planning.

Typical use:

- internal AI expert platform;
- controlled knowledge reuse;
- enterprise workflow standardization.

---

# 7. Final Principle

> Adoption improves when the system speaks clearly to the needs of each user type.
