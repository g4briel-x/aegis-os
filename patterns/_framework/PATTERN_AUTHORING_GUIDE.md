# Pattern Authoring Guide

Version: 0.1.0  
Status: Foundation Draft

---

# 1. Purpose

This guide explains how to create a useful Aegis OS Pattern.

---

# 2. Authoring Process

Use this sequence:

```text
1. Identify recurring problem
2. Define usage context
3. Explain forces and constraints
4. Describe recommended solution
5. Document trade-offs
6. Document risks and failure modes
7. Add implementation guidance
8. Add validation checklist
9. Add examples
10. Link related skills and playbooks
```

---

# 3. Writing Rules

A Pattern should:

- explain why the solution exists;
- define when it applies;
- define when it does not apply;
- include trade-offs;
- include examples;
- avoid pretending one solution fits all cases.

---

# 4. Common Mistakes

Avoid:

```text
Generic advice
Missing trade-offs
No examples
No validation checklist
Overly broad pattern
No context limits
No related assets
```

---

# 5. Final Principle

> Good Pattern writing helps future teams decide faster without hiding complexity.
