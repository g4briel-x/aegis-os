# Pattern Review Checklist

Version: 0.1.0  
Status: Foundation Draft

---

# 1. Purpose

This checklist helps reviewers evaluate a Pattern before approval.

---

# 2. Review Checklist

```text
[ ] Pattern id is unique
[ ] Pattern name is clear
[ ] Problem is recurring
[ ] Context is explicit
[ ] Recommended use is clear
[ ] Avoid-use cases are included
[ ] Solution is actionable
[ ] Trade-offs are honest
[ ] Risks are realistic
[ ] Examples are useful
[ ] Checklist is practical
[ ] Related assets are linked
[ ] Metadata is valid
```

---

# 3. Reviewer Decision

Possible decisions:

```text
approve
request_changes
reject
merge_as_draft
deprecate_duplicate
```

---

# 4. Final Principle

> Pattern review protects the repository from reusable confusion.
