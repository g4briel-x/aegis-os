# SaaS Onboarding Flow — Checklists

Version: 0.1.0  
Status: Draft

---

# 1. Context Fit Checklist

```text
[ ] Product has new users
[ ] First value requires action
[ ] Activation can be measured
[ ] Setup is needed
[ ] Empty states exist
[ ] User drop-off matters
```

---

# 2. Activation Checklist

```text
[ ] Activation moment is defined
[ ] Activation event is trackable
[ ] Activation connects to core value
[ ] Time to activation can be measured
[ ] Drop-off before activation can be analyzed
[ ] Success threshold is defined
```

---

# 3. Flow Checklist

```text
[ ] Signup is clear
[ ] Setup questions are minimal
[ ] First object creation is obvious
[ ] First workflow is guided
[ ] Next best action is visible
[ ] Skip path exists where appropriate
```

---

# 4. Empty State Checklist

```text
[ ] Empty state explains purpose
[ ] Empty state explains benefit
[ ] Primary action is visible
[ ] Example or template is available if useful
[ ] Copy is specific
[ ] Empty state is not generic filler
```

---

# 5. Measurement Checklist

```text
[ ] Signup completion tracked
[ ] Workspace creation tracked
[ ] First object creation tracked
[ ] First workflow completion tracked
[ ] Activation completion tracked
[ ] Abandonment step tracked
```

---

# 6. Test Checklist

```text
[ ] New user can complete onboarding
[ ] User understands next action
[ ] Empty states guide action
[ ] Setup does not block unnecessarily
[ ] Activation event fires correctly
[ ] Mobile or small-screen flow is usable if relevant
```

---

# 7. Final Principle

> Onboarding checklists should prove that new users can reach value without handholding.
