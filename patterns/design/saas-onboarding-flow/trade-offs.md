# SaaS Onboarding Flow — Trade-Offs

Version: 0.1.0  
Status: Draft

---

# 1. Benefits

This Pattern improves:

```text
activation rate
time to value
user clarity
trial conversion
product learning
support efficiency
retention signals
```

---

# 2. Costs

This Pattern adds:

```text
UX design effort
event tracking work
state management
personalization logic
copywriting effort
experimentation overhead
```

---

# 3. Friction Trade-Off

Asking questions can improve personalization but increase drop-off.

Mitigation:

```text
ask only necessary questions
make questions optional when possible
explain why information is needed
delay non-critical questions
```

---

# 4. Guidance Trade-Off

Guidance helps new users but can annoy experienced users.

Mitigation:

```text
allow skipping
remember completed steps
offer contextual help
avoid blocking core navigation
```

---

# 5. Empty State Trade-Off

Helpful empty states improve action but can feel promotional.

Mitigation:

```text
keep copy direct
show real examples
prioritize one primary action
avoid generic marketing text
```

---

# 6. Main Risks

Key risks:

```text
onboarding too long
activation metric poorly chosen
setup data unused
role-specific paths too complex
progress indicators meaningless
onboarding not connected to retention
```

---

# 7. Mitigations

Mitigate with:

- user testing;
- analytics;
- activation definition;
- drop-off review;
- copy review;
- progressive disclosure;
- onboarding experiments.

---

# 8. Final Principle

> Onboarding trade-offs should favor first value over feature exposure.
