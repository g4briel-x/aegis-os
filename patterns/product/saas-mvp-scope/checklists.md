# SaaS MVP Scope — Checklists

Version: 0.1.0  
Status: Draft

---

# 1. Context Fit Checklist

```text
[ ] Product is early-stage
[ ] Scope needs focus
[ ] Customer problem is not fully validated
[ ] Team needs fast learning
[ ] MVP launch or pilot is planned
[ ] Success metric is needed
```

---

# 2. Segment Checklist

```text
[ ] One primary segment selected
[ ] Buyer or user identified
[ ] Pain point is specific
[ ] Segment is reachable
[ ] Segment has urgency
[ ] Segment can provide feedback quickly
```

---

# 3. Workflow Checklist

```text
[ ] One core workflow defined
[ ] Workflow has clear start
[ ] Workflow has clear end
[ ] Activation moment is defined
[ ] Workflow can be completed in MVP
[ ] Workflow produces measurable value
```

---

# 4. Feature Inclusion Checklist

```text
[ ] Feature supports core workflow
[ ] Feature supports activation
[ ] Feature supports trust baseline
[ ] Feature supports learning metric
[ ] Feature supports pilot or payment validation
[ ] Feature is not speculative
```

---

# 5. Non-Scope Checklist

```text
[ ] Nice-to-have features excluded
[ ] Advanced automation delayed
[ ] Non-critical integrations delayed
[ ] Advanced analytics delayed
[ ] Multi-segment features delayed
[ ] Future roadmap items documented separately
```

---

# 6. Launch Learning Checklist

```text
[ ] Success metric defined
[ ] Feedback channel defined
[ ] Pilot or beta audience defined
[ ] Activation tracked
[ ] Willingness-to-pay signal planned
[ ] Review cadence defined
```

---

# 7. Final Principle

> MVP scope checklists keep the product small, valuable and testable.
