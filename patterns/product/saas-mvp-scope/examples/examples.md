# 1. Example — Audiovisual Financing SaaS

## Context

A platform helps creators prepare film, series and documentary projects for financing.

## Target Segment

```text
Independent producers preparing a project package for financing discussions.
```

## Core Workflow

```text
Create project
Complete project profile
Upload pitch materials
Submit for readiness review
Receive readiness feedback
```

## MVP Features

```text
user account
project profile
document upload
submission status
review dashboard
feedback notes
basic notification
```

## Non-Scope

```text
investor marketplace
automated funding matching
advanced analytics
public project pages
payment subscriptions
AI-generated pitch decks
mobile app
```

## Success Metric

```text
At least 10 producers submit complete project packages during beta.
```

# 2. Example — B2B Workflow SaaS

## Target Segment

```text
Small operations teams managing recurring approval workflows.
```

## Core Workflow

```text
Create workflow
Invite reviewer
Submit request
Approve or reject
Track status
```

## Non-Scope

```text
complex automation builder
third-party integrations
advanced reporting
enterprise SSO
```

---

# 3. Example — AI Document Review SaaS

## Target Segment

```text
Small legal teams reviewing standard contracts.
```

## Core Workflow

```text
Upload document
Run AI review
View flagged clauses
Export summary
```

## Non-Scope

```text
custom model training
multi-language review
document collaboration
enterprise audit console
```

# 4. Example — MVP Inclusion Decision

```text
Feature: advanced dashboard analytics
Decision: exclude from MVP
Reason: does not directly support first activation moment
Future: revisit after 20 active beta users
```

# 5. Final Principle

> Examples show that MVP scope must be narrow enough to ship and meaningful enough to validate.
