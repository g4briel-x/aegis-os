# Aegis OS — Playbook Authoring Guide

Version: 0.1  
Status: Playbook Guide

---

# 1. Introduction

This guide explains how to write a high-quality Aegis OS Playbook.

---

# 2. Authoring Philosophy

Aegis OS follows this principle:

> A Playbook should encode the steps of repeatable expert execution, not just describe the topic.

---

# 3. Authoring Steps

Recommended process:

1. Identify the recurring scenario.
2. Define the trigger.
3. Define required inputs.
4. Define expected outputs.
5. Break the workflow into steps.
6. Add decision points.
7. Add validation checklists.
8. Define completion criteria.
9. Add examples.
10. Run quality review.

---

# 4. Writing Rules

A Playbook should be:

- action-oriented;
- sequential;
- specific;
- decision-aware;
- validation-oriented;
- practical.

Avoid:

- vague steps;
- missing triggers;
- missing outputs;
- unclear completion criteria;
- procedures that cannot be executed.

---

# 5. Final Principle

> The best Playbooks are written like procedures that a competent team can execute under pressure.
