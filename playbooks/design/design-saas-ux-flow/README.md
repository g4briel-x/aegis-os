---

# 1. Purpose

The Design SaaS UX Flow Playbook provides a structured procedure for turning product requirements into clear, usable SaaS user flows.

It helps align product intent, user goals, screens, states, permissions, edge cases and handoff expectations before visual design or development begins.

---

# 2. Trigger

Use this Playbook when:

- a PRD needs to become a UX flow;
- a SaaS feature needs screen sequencing;
- a user journey is unclear;
- a dashboard, form, portal or workflow needs design structure;
- product and engineering need shared screen logic;
- a feature has too many states or roles;
- a design handoff needs stronger clarity.

---

# 3. Scope

This Playbook covers:

- user goal definition;
- entry and exit points;
- task flow mapping;
- screen inventory;
- state definition;
- role and permission impact;
- edge cases;
- UX risk review;
- design handoff outputs.

This Playbook does not replace full visual UI design, but it creates the flow foundation for wireframes, prototypes and implementation.
