# Design SaaS UX Flow — Checklists

Version: 0.1.0  
Status: Premium Draft

---

# 1. Flow Foundation Checklist

```text
[ ] Primary user defined
[ ] User goal defined
[ ] Success condition defined
[ ] Entry points identified
[ ] Exit points identified
[ ] Related PRD or requirement referenced
[ ] Constraints documented
```

---

# 2. Screen Checklist

```text
[ ] Required screens listed
[ ] Screen purpose defined
[ ] Primary action identified
[ ] Secondary actions identified
[ ] Navigation behavior defined
[ ] Empty states considered
[ ] Error states considered
[ ] Success states considered
```

---

# 3. Role and Permission Checklist

```text
[ ] Roles identified
[ ] Allowed actions defined
[ ] Restricted actions defined
[ ] Permission denied state defined
[ ] Admin behavior defined if relevant
[ ] Tenant or ownership boundaries considered
```

---

# 4. Usability Checklist

```text
[ ] Flow has minimal necessary steps
[ ] Labels are clear
[ ] User receives feedback after actions
[ ] Errors explain recovery path
[ ] Forms avoid unnecessary fields
[ ] Defaults are considered
[ ] Mobile or responsive needs considered
[ ] Accessibility needs considered
```

---

# 5. Handoff Checklist

```text
[ ] Flow summary completed
[ ] Screens documented
[ ] States documented
[ ] Edge cases documented
[ ] Permissions documented
[ ] Open questions listed
[ ] Design or engineering owner identified
```

---

# 6. Final Principle

> UX flow checklists make invisible user experience risks visible before design polish.
