# 1. Example — Audiovisual Project Submission Flow

## Trigger

A SaaS platform needs a flow where creators submit film, series or documentary projects for financing review.

## Expected Execution

The Playbook should guide the user to:

- define creator goal;
- map project submission steps;
- identify required screens;
- define draft, submitted and review states;
- define upload errors;
- define reviewer permission behavior;
- produce UX handoff.

## Expected Output

```text
UX flow summary
User journey
Screen inventory
Screen states
Edge cases
Permission behavior
Handoff checklist
```

# 2. Example — SaaS Dashboard Onboarding Flow

## Trigger

A new user needs to complete onboarding before accessing a SaaS dashboard.

## Expected Execution

The Playbook should guide the user to:

- define entry point after signup;
- map onboarding questions;
- define skip behavior;
- define empty dashboard state;
- define completion confirmation;
- reduce friction.

# 3. Example — Billing Settings Flow

## Trigger

A SaaS user needs to update billing information.

## Expected Execution

The Playbook should guide the user to:

- identify account owner role;
- define payment update screen;
- define validation and error states;
- define confirmation state;
- flag security-sensitive behavior.

# 4. Example — Admin User Invitation Flow

## Trigger

An admin needs to invite team members into a SaaS workspace.

## Expected Execution

The Playbook should guide the user to:

- define admin permissions;
- map invite creation;
- define pending invite state;
