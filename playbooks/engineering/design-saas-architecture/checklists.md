# Design SaaS Architecture — Checklists

Version: 0.1.0  
Status: Premium Draft

---

# 1. Architecture Context Checklist

```text
[ ] Product purpose defined
[ ] MVP scope available
[ ] User roles identified
[ ] Business constraints captured
[ ] Technical constraints captured
[ ] Expected scale estimated
[ ] Open assumptions listed
```

---

# 2. Component Checklist

```text
[ ] Frontend component defined
[ ] Backend/API component defined
[ ] Database component defined
[ ] Auth component defined
[ ] File storage considered if needed
[ ] Background jobs considered if needed
[ ] Admin or operations surface considered
[ ] External integrations listed
```

---

# 3. Data Checklist

```text
[ ] Core entities identified
[ ] Entity relationships described
[ ] Ownership model defined
[ ] Tenant boundary considered
[ ] Sensitive data identified
[ ] Audit needs considered
[ ] Backup and recovery considered
```

---

# 4. API and Auth Checklist

```text
[ ] API style defined
[ ] Key endpoints listed
[ ] Request validation considered
[ ] Error handling approach defined
[ ] Authentication model defined
[ ] Authorization model defined
[ ] Role permissions considered
[ ] Public endpoints reviewed
```

---

# 5. Infrastructure Checklist

```text
[ ] Hosting approach defined
[ ] Environments defined
[ ] Deployment approach defined
[ ] Secrets management defined
[ ] Configuration approach defined
[ ] Logging and monitoring defined
[ ] Backup approach defined
[ ] Rollback approach considered
```

---

# 6. Architecture Quality Checklist

```text
[ ] Security risks reviewed
[ ] Scalability risks reviewed
[ ] Performance risks reviewed
[ ] Operational risks reviewed
[ ] Cost risks reviewed
[ ] Implementation sequence defined
[ ] Open decisions documented
```

---

# 7. Final Principle

> Architecture checklists make sure the system can be built, operated and evolved.
