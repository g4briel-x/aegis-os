# 1. Example — Audiovisual Financing SaaS

## Trigger

A SaaS platform helps creators package film, series and documentary projects and move them toward financing.

## Expected Execution

The Playbook should guide the user to:

- define creator, reviewer, admin and investor roles;
- identify project, document, pitch, review and funding entities;
- define project submission API;
- define file storage and access control;
- define tenant or workspace boundaries;
- create MVP-friendly architecture.

## Expected Output

```text
Architecture summary
Component map
Data model outline
API model
Auth model
Infrastructure plan
Risk register
Implementation sequence
```

# 2. Example — B2B Client Portal SaaS

## Trigger

A company needs a secure client portal for projects, invoices and documents.

## Expected Execution

The Playbook should guide the user to:

- define client and internal admin roles;
- separate tenant data;
- design document storage;
- define project and invoice APIs;
- include audit logs and permission rules.

---

# 3. Example — AI Assistant SaaS

## Trigger

A team wants to build an AI assistant with chat, files, memory and tool use.

## Expected Execution

The Playbook should guide the user to:

- define chat, file, memory and tool components;
- identify prompt injection and data risks;
- define storage model;
- define access boundaries;
- include observability and cost controls.

---

# 4. Example — Marketplace SaaS

## Trigger

A SaaS marketplace connects service providers and customers.

## Expected Execution

The Playbook should guide the user to:

- define account, listing, booking and payment components;
- identify trust and moderation needs;
- define transaction boundaries;
- include security and dispute considerations.

# 5. Final Principle

> Examples show that SaaS architecture must connect product value, technical structure and operational reality.
