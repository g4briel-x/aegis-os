# Implement Feature From PRD — Checklists

Version: 0.1.0  
Status: Premium Draft

---

# 1. Implementation Readiness Checklist

```text
[ ] PRD reviewed
[ ] Goals and non-goals understood
[ ] User stories reviewed
[ ] Acceptance criteria reviewed
[ ] Open questions listed
[ ] Engineering assumptions documented
[ ] Scope boundaries confirmed
```

---

# 2. Technical Planning Checklist

```text
[ ] Affected frontend areas identified
[ ] Affected backend areas identified
[ ] API impact identified
[ ] Data impact identified
[ ] Auth and permission impact identified
[ ] Integration impact identified
[ ] Configuration impact identified
[ ] Documentation impact identified
```

---

# 3. Implementation Checklist

```text
[ ] Technical tasks defined
[ ] Data changes implemented if needed
[ ] API changes implemented if needed
[ ] UI changes implemented if needed
[ ] Validation logic implemented
[ ] Error handling implemented
[ ] Permission checks implemented
[ ] Edge cases handled
```

---

# 4. Testing Checklist

```text
[ ] Unit tests added or updated
[ ] Integration tests added or updated
[ ] API tests added or updated if needed
[ ] UI tests added or updated if needed
[ ] Permission tests added if needed
[ ] Acceptance criteria verified
[ ] Regression risk reviewed
```

---

# 5. Pull Request Checklist

```text
[ ] PR summary written
[ ] Requirements linked
[ ] Changed areas listed
[ ] Test evidence included
[ ] Screenshots included if UI changed
[ ] Migration notes included if needed
[ ] Security notes included if needed
[ ] Rollback notes included if needed
```

---

# 6. Final Principle

> Implementation checklists protect code quality, product intent and release safety.
