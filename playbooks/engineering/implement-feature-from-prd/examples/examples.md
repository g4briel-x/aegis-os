# 1. Example — Audiovisual Project Submission Feature

## Trigger

A PRD defines a feature where creators can submit film, series or documentary projects for financing review.

## Expected Execution

The Playbook should guide the engineer to:

- review submission requirements;
- identify form, API, database and file storage changes;
- define project status states;
- implement role permissions;
- add validation and tests;
- prepare PR and release notes.

## Expected Output

```text
Implementation plan
Task breakdown
API and data change summary
Test plan
Pull request plan
Release handoff notes
```

# 2. Example — Client Portal Invoice View

## Trigger

A PRD defines a client portal screen for viewing invoices.

## Expected Execution

The Playbook should guide the engineer to:

- identify permissions;
- create API endpoint;
- define invoice response fields;
- build frontend view;
- test tenant isolation;
- document security notes.

# 3. Example — Admin User Invitation

## Trigger

A PRD defines an admin feature to invite team members.

## Expected Execution

The Playbook should guide the engineer to:

- define invite data model;
- implement email invitation flow;
- handle expired invitations;
- prevent duplicate invites;
- add audit logs;
- test role restrictions.
