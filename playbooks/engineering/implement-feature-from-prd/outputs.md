```text
Feature:
PRD reference:
Technical approach:
Affected areas:
Assumptions:
Risks:
```

Purpose:

Convert product requirements into engineering tasks.

Required sections:

```text
Task:
Category:
Owner:
Dependency:
Acceptance note:
Status:
```

Purpose:

Document what parts of the system will change.

Required sections:

```text
Component:
Change:
Reason:
Risk:
Validation:
```

---

Purpose:

Document API and persistence changes.

Required sections:

```text
Area:
Change:
Contract or schema:
Migration needed:
Compatibility notes:
```

---

# 5. Test Plan

Purpose:

Define verification before merge and release.

Required sections:

```text
Requirement:
Test type:
Test case:
Expected result:
Status:
```

---

# 6. Pull Request Plan

Purpose:

Prepare the implementation for review.

Required sections:

```text
PR title:
Summary:
Changed areas:
Testing:
Risks:
Reviewers:
Release notes:
```

---

# 7. Release Handoff Notes

Purpose:

Prepare operations or release owner for deployment.

Required sections:

```text
Feature:
Deployment notes:
Migration notes:
Rollback notes:
Monitoring notes:
Support notes:
```

# 8. Final Principle

> Implementation outputs must make the work reviewable, testable and releasable.
