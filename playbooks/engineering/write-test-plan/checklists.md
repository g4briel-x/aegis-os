# Write Test Plan — Checklists

Version: 0.1.0  
Status: Premium Draft

---

# 1. Requirement Checklist

```text
[ ] PRD or requirement reviewed
[ ] User stories reviewed
[ ] Acceptance criteria reviewed
[ ] Goals and non-goals understood
[ ] Open questions listed
[ ] Requirement gaps identified
[ ] Testability reviewed
```

---

# 2. Test Scope Checklist

```text
[ ] Included features listed
[ ] Included workflows listed
[ ] Affected components listed
[ ] Excluded areas documented
[ ] Critical paths identified
[ ] Risk areas identified
[ ] Test owner identified
```

---

# 3. Test Case Checklist

```text
[ ] Functional tests written
[ ] Edge cases written
[ ] Error cases written
[ ] Permission cases written if needed
[ ] Regression cases written
[ ] Smoke tests defined if needed
[ ] Expected results are clear
```

---

# 4. Environment and Data Checklist

```text
[ ] Test environment defined
[ ] Test users defined
[ ] Test roles defined
[ ] Test records defined
[ ] Mock services identified
[ ] Seed data defined if needed
[ ] Sensitive data avoided
```

---

# 5. Release Validation Checklist

```text
[ ] Critical tests identified
[ ] Test evidence required
[ ] Owners assigned
[ ] Blockers listed
[ ] Risks documented
[ ] Final validation decision defined
[ ] Follow-up actions captured
```

---

# 6. Final Principle

> Test plan checklists keep quality from depending on memory and assumptions.
