Version: 0.1.0

# 1. Example — Audiovisual Financing SaaS MVP

## Trigger

A SaaS platform for film, series and documentary creators has MVP scope, PRD, UX flow and architecture ready.

## Expected Execution

The Playbook should guide the team to:

- split work into product, design, frontend, backend, database, infrastructure and security;
- create milestones for foundation, project submission, review workflow and launch readiness;
- identify dependencies between auth, file storage, project data and reviewer access;
- define delivery risks;
- prepare release readiness.

## Expected Output

```text
Delivery plan
Workstream map
Milestone roadmap
Prioritized backlog
Dependency map
Ownership matrix
Risk register
Launch readiness plan
```

# 2. Example — Client Portal MVP

## Trigger

A business wants to launch a client portal with projects, invoices and documents.

## Expected Execution

The Playbook should guide the team to:

- sequence account setup, project records, file access and invoice visibility;
- include permission review;
- create QA and release workstreams;
- define launch checklist.

# 3. Example — AI Assistant MVP

## Trigger

A team wants to build an AI assistant with chat, files and memory.

## Expected Execution

The Playbook should guide the team to:

- define foundation milestone for auth and storage;
- define core AI interaction milestone;
- include safety and observability work;
- identify cost and latency risks;
- define alpha release.

# 4. Example — Marketplace MVP

## Trigger

A SaaS marketplace needs first delivery planning.

## Expected Execution

The Playbook should guide the team to:
