# 1. Purpose

The Create Runbook Playbook provides a structured procedure for writing operational runbooks that teams can use during incidents, deployments, maintenance, support escalations and recurring operations.

It helps convert operational knowledge into clear steps, triggers, inputs, commands, checks, rollback actions, escalation paths and verification criteria.

---

# 2. Trigger

Use this Playbook when:

- an operational process must be documented;
- a recurring task depends on tribal knowledge;
- incident response needs repeatable steps;
- deployment or rollback actions need clarity;
- production support needs a standard guide;
- onboarding new operators requires procedure documentation;
- a postmortem identifies missing runbooks.

---

# 3. Scope

This Playbook covers:

- runbook purpose;
- trigger conditions;
- prerequisites;
- required access;
- step-by-step procedure;
- commands and checks;
- decision points;
- rollback and recovery;
- escalation path;
- verification;
- maintenance ownership.

This Playbook does not replace automated workflows, but it documents safe operational execution until automation exists.
