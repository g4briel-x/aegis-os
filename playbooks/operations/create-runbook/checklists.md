# Create Runbook — Checklists

Version: 0.1.0  
Status: Premium Draft

---

# 1. Runbook Scope Checklist

```text
[ ] Runbook name defined
[ ] Purpose defined
[ ] Scope defined
[ ] Non-scope defined
[ ] Intended operator identified
[ ] Affected systems listed
[ ] Expected result defined
```

---

# 2. Trigger Checklist

```text
[ ] Trigger event defined
[ ] Alert trigger defined if relevant
[ ] Manual trigger defined if relevant
[ ] Non-trigger scenarios defined
[ ] Approval requirement defined
[ ] Emergency usage clarified
```

---

# 3. Safety Checklist

```text
[ ] Required access listed
[ ] Destructive actions identified
[ ] Customer impact reviewed
[ ] Backup need reviewed
[ ] Maintenance window need reviewed
[ ] Rollback need reviewed
[ ] Approval gate added if needed
```

---

# 4. Procedure Checklist

```text
[ ] Steps are ordered
[ ] Commands are explicit
[ ] Expected result is included
[ ] Evidence is defined
[ ] Failure path is defined
[ ] Decision points are included
[ ] No secret values are exposed
```

---

# 5. Escalation Checklist

```text
[ ] Technical owner listed
[ ] Incident owner listed if relevant
[ ] Security owner listed if relevant
[ ] Communication channel listed
[ ] Escalation thresholds defined
[ ] Support or product owner listed if needed
```

---

# 6. Verification Checklist

```text
[ ] Success criteria defined
[ ] Validation commands listed
[ ] Dashboard or log checks listed
[ ] Post-execution note required
[ ] Maintenance owner assigned
[ ] Review cadence defined
```

---

# 7. Final Principle

> Runbook checklists help operators act consistently when speed and accuracy both matter.
