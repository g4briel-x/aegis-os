Version: 0.1.0  
Status: Premium Draft  
Domain: Operations

---

# 1. Purpose

The Run Postmortem Review Playbook provides a structured procedure for reviewing incidents, outages, security events, failed releases and major delivery failures after resolution.

It helps teams reconstruct the timeline, identify root cause, separate contributing factors, capture lessons, assign prevention actions and improve systems without blame.

---

# 2. Trigger

Use this Playbook when:

- a production incident has been resolved;
- a security incident has been contained;
- a release failed or required rollback;
- a critical workflow was unavailable;
- a customer-impacting bug occurred;
- a major CI/CD or deployment failure delayed delivery;
- leadership or the team needs a clear incident review.

---

# 3. Scope

This Playbook covers:

- incident summary;
- timeline reconstruction;
- impact review;
- root cause analysis;
- contributing factors;
- response effectiveness;
- detection gaps;
- prevention actions;
- ownership and follow-up tracking.

This Playbook is not a blame process. It is a learning and system-improvement process.
