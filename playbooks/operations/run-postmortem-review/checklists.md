# Run Postmortem Review — Checklists

Version: 0.1.0  
Status: Premium Draft

---

# 1. Review Preparation Checklist

```text
[ ] Incident summary available
[ ] Severity recorded
[ ] Affected services listed
[ ] Review owner assigned
[ ] Participants identified
[ ] Evidence sources listed
[ ] Meeting or review format selected
```

---

# 2. Timeline Checklist

```text
[ ] First signal recorded
[ ] Detection time recorded
[ ] Acknowledgment time recorded
[ ] First mitigation recorded
[ ] Recovery time recorded
[ ] Communication events recorded
[ ] Final resolution recorded
```

---

# 3. Impact Checklist

```text
[ ] User impact described
[ ] Workflow impact described
[ ] Business impact described
[ ] Data impact reviewed
[ ] Security impact reviewed
[ ] Support impact reviewed
[ ] Duration recorded
```

---

# 4. Root Cause Checklist

```text
[ ] Symptoms separated from cause
[ ] Trigger identified
[ ] Root cause supported by evidence
[ ] Contributing factors listed
[ ] Alternative causes considered
[ ] Unknowns documented
```

---

# 5. Response Review Checklist

```text
[ ] Detection quality reviewed
[ ] Triage quality reviewed
[ ] Escalation reviewed
[ ] Communication reviewed
[ ] Mitigation reviewed
[ ] Recovery verification reviewed
[ ] Monitoring gaps identified
```

---

# 6. Prevention Checklist

```text
[ ] Prevention actions defined
[ ] Owners assigned
[ ] Priorities assigned
[ ] Due dates assigned
[ ] Verification method defined
[ ] Follow-up tracker created
```

---

# 7. Final Principle

> Postmortem checklists keep the review focused on truth, learning and prevention.
