---

# 1. Purpose

The Create PRD Playbook provides a structured procedure for turning a product idea, feature request or business need into a clear Product Requirements Document.

It helps align product, design, engineering, QA, security and stakeholders before implementation begins.

---

# 2. Trigger

Use this Playbook when:

- a new feature needs product definition;
- an MVP feature needs requirements;
- a stakeholder request must be clarified;
- engineering needs implementation-ready requirements;
- design needs user flows and screen expectations;
- a roadmap item needs execution detail;
- acceptance criteria are missing or vague.

---

# 3. Scope

This Playbook covers:

- context definition;
- problem statement;
- goals and non-goals;
- user personas;
- user stories;
- functional requirements;
- non-functional requirements;
- acceptance criteria;
- metrics;
- risks;
- dependencies;
- open questions.

This Playbook does not replace technical architecture or UX/UI design, but it creates the product foundation those teams need.
