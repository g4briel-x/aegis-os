# Create PRD — Checklists

Version: 0.1.0  
Status: Premium Draft

---

# 1. PRD Foundation Checklist

```text
[ ] Product context documented
[ ] Business objective defined
[ ] Problem statement clear
[ ] Target users identified
[ ] Evidence or assumptions documented
[ ] Stakeholders identified
```

---

# 2. Scope Checklist

```text
[ ] Goals defined
[ ] Non-goals defined
[ ] Included workflows listed
[ ] Excluded items documented
[ ] Release boundary clear
[ ] Dependencies identified
```

---

# 3. Requirement Quality Checklist

```text
[ ] User stories written
[ ] Functional requirements clear
[ ] Non-functional requirements considered
[ ] Permissions considered
[ ] States and errors considered
[ ] Data requirements considered
[ ] Requirements are testable
```

---

# 4. Acceptance Checklist

```text
[ ] Acceptance criteria are specific
[ ] Acceptance criteria are testable
[ ] Success and failure states included
[ ] Edge cases considered
[ ] QA can verify requirements
[ ] Product owner can approve outcome
```

---

# 5. Risk Checklist

```text
[ ] Product risks identified
[ ] Technical risks identified
[ ] UX risks identified
[ ] Security risks identified
[ ] Dependencies listed
[ ] Open questions captured
[ ] Decision owners identified
```

---

# 6. Final Principle

> PRD checklists protect teams from building from vague agreement.
