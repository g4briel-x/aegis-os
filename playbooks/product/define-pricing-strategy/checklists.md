# Define Pricing Strategy — Checklists

Version: 0.1.0  
Status: Premium Draft

---

# 1. Customer Segment Checklist

```text
[ ] Target customers listed
[ ] Buyer identified
[ ] User identified
[ ] Decision maker identified
[ ] Use cases defined
[ ] Urgency level reviewed
[ ] Willingness-to-pay assumptions listed
```

---

# 2. Value Checklist

```text
[ ] Main pain points identified
[ ] Value drivers listed
[ ] Business impact estimated
[ ] Differentiators identified
[ ] Upgrade triggers identified
[ ] Price sensitivity considered
[ ] Customer objections anticipated
```

---

# 3. Pricing Metric Checklist

```text
[ ] Pricing metric selected
[ ] Metric aligns with value
[ ] Metric is easy to understand
[ ] Metric aligns with cost structure
[ ] Metric is technically billable
[ ] Metric does not punish healthy usage
[ ] Metric can scale with growth
```

---

# 4. Packaging Checklist

```text
[ ] Tiers defined
[ ] Features assigned to tiers
[ ] Limits defined
[ ] Add-ons considered
[ ] Free trial or freemium decision made
[ ] Enterprise option reviewed
[ ] Upgrade path is clear
```

---

# 5. Economics Checklist

```text
[ ] Infrastructure cost reviewed
[ ] Support cost reviewed
[ ] Onboarding cost reviewed
[ ] AI or API cost reviewed if relevant
[ ] Payment fees considered
[ ] Gross margin target reviewed
[ ] Cost-to-serve risk documented
```

---

# 6. Validation Checklist

```text
[ ] Pricing assumptions documented
[ ] Validation experiment selected
[ ] Interview questions prepared
[ ] Success criteria defined
[ ] Objection tracking planned
[ ] Review date scheduled
[ ] Owner assigned
```

---

# 7. Final Principle

> Pricing checklists prevent teams from launching prices based only on instinct.
