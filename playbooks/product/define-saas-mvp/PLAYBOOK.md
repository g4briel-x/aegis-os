# Define SaaS MVP — Playbook Definition

Version: 0.1.0  
Status: Premium Draft

---

# 1. Purpose

Guide a structured process for defining the first useful version of a SaaS product.

---

# 2. Trigger

A SaaS concept, feature set or business idea needs to be reduced into a focused MVP.

---

# 3. Inputs

Useful inputs include:

- product idea;
- target customer;
- user problem;
- business goal;
- market context;
- feature list;
- technical constraints;
- budget or timeline;
- existing research;
- competitor notes.

---

# 4. Outputs

Expected outputs include:

- MVP definition;
- target user summary;
- problem statement;
- value proposition;
- must-have feature list;
- excluded feature list;
- success metrics;
- launch risks;
- first roadmap sequence.

---

# 5. Execution Summary

```text
1. Clarify the SaaS idea
2. Identify target users
3. Define the core problem
4. Define the value proposition
5. Identify the core workflow
6. List candidate features
7. Separate must-have from later features
8. Define MVP success metrics
9. Identify risks and assumptions
10. Produce the MVP scope and first roadmap
```

---

# 6. Completion Criteria

The Playbook is complete when:

- the target user is clear;
- the problem is specific;
- the MVP has a small coherent scope;
- excluded features are documented;
- success metrics are defined;
- risks and assumptions are visible;
- the next delivery step is clear.

---

# 7. Escalation or Fallback

Escalate to additional Skills when:

- technical feasibility is unclear;
- UX flow is complex;
- pricing requires deeper work;
- market validation is weak;
- security, data or compliance risks are high.

---

# 8. Final Principle

> The MVP is not the smallest feature list. It is the smallest product that can validate the core value.
