Version: 0.1.0  
Status: Premium Draft  
Domain: Product  
Category: SaaS Product Definition

---

# 1. Purpose

The Define SaaS MVP Playbook provides a structured procedure for turning a SaaS idea into a focused, realistic and testable Minimum Viable Product.

This Playbook prevents early product overload by forcing clarity around the target user, core problem, minimum workflow, success metrics, risks and excluded features.

---

# 2. Trigger

Use this Playbook when:

- a new SaaS idea needs MVP definition;
- an existing idea has too many features;
- a team needs launch scope;
- product, design and engineering need alignment;
- a founder needs a first build plan;
- a roadmap needs to start with a focused first version.

---

# 3. Scope

This Playbook covers:

- target user definition;
- problem framing;
- value proposition;
- MVP scope;
- feature prioritization;
- success metrics;
- launch risks;
- first roadmap sequence.

This Playbook does not replace full product discovery, technical architecture or UX/UI design, but it creates the input for those activities.
