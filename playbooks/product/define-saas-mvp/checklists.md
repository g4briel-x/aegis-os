# Define SaaS MVP — Checklists

Version: 0.1.0  
Status: Premium Draft

---

# 1. MVP Clarity Checklist

```text
[ ] Product idea summarized in one sentence
[ ] Primary target user defined
[ ] Core problem stated clearly
[ ] Current workaround identified
[ ] Value proposition defined
[ ] Business goal identified
```

---

# 2. Scope Checklist

```text
[ ] Core workflow defined
[ ] Candidate features listed
[ ] Must-have features selected
[ ] Nice-to-have features removed
[ ] Excluded features documented
[ ] MVP can be explained simply
```

---

# 3. Validation Checklist

```text
[ ] Key assumptions listed
[ ] Discovery evidence identified
[ ] Success metrics defined
[ ] Willingness-to-pay question considered
[ ] Launch feedback plan defined
[ ] Failure signals defined
```

---

# 4. Delivery Readiness Checklist

```text
[ ] First roadmap sequence defined
[ ] Technical feasibility reviewed
[ ] UX complexity reviewed
[ ] Risks documented
[ ] Dependencies listed
[ ] Next execution step clear
```

---

# 5. Final Principle

> MVP checklists protect the product from becoming a first version of everything.
