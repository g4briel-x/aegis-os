# 1. Example — Audiovisual Financing SaaS

## Trigger

A founder wants to build a SaaS platform that helps film, series and documentary projects move from idea to financing.

## Expected Execution

The Playbook should guide the user to:

- identify creators or producers as first users;
- define the core problem;
- select the minimum workflow;
- exclude advanced marketplace features at first;
- define success metrics;
- create a phased MVP roadmap.

## Expected Output

```text
Target user
Problem statement
Value proposition
Core workflow
MVP scope
Excluded features
Success metrics
Risks
First roadmap
```

# 2. Example — Freelancer Management SaaS

## Trigger

A user wants to create a SaaS for freelancers to manage clients, invoices and projects.

## Expected Execution

The Playbook should guide the user to:

- focus on one target segment;
- identify the core workflow;
- include clients, projects and invoices only if necessary;
- exclude advanced analytics and automation;
- define activation success.

---

# 3. Example — AI Assistant SaaS

## Trigger

A team wants to build an AI assistant SaaS with chat, memory, tools, files, analytics and team collaboration.

## Expected Execution

The Playbook should guide the user to:

- define the first job to be done;
- reduce AI features to the core value;
- exclude non-essential collaboration or analytics;
- define safety and trust requirements;
- identify technical feasibility risks.

---

# 4. Example — Internal Tool SaaS

## Trigger

A company wants to convert an internal spreadsheet workflow into a SaaS-like internal tool.

## Expected Execution

The Playbook should guide the user to:

- define current workflow;
- identify pain points;
- select the minimum digital workflow;
- define data fields;
- exclude unnecessary dashboards;
- define success as time saved or error reduction.

# 5. Final Principle

> Examples show that a strong MVP is not a small dream. It is a focused proof of value.
