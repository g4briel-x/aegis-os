# Define SaaS MVP — Steps

Version: 0.1.0  
Status: Premium Draft

---

# 1. Step 1 — Clarify the SaaS Idea

Summarize the product in one clear sentence.

Capture:

- what the product does;
- who it serves;
- what problem it solves;
- why it matters.

Output:

```text
One-sentence SaaS concept
```

---

# 2. Step 2 — Identify the Target User

Define the first user segment.

Clarify:

- user type;
- industry or context;
- current workflow;
- pain level;
- buying influence;
- urgency.

Output:

```text
Primary target user profile
```

---

# 3. Step 3 — Define the Core Problem

State the problem without jumping to features.

A strong problem statement should include:

- user;
- situation;
- pain;
- consequence;
- current workaround.

Output:

```text
Core problem statement
```

---

# 4. Step 4 — Define the Value Proposition

Explain why the SaaS should exist.

Clarify:

- main promise;
- measurable benefit;
- difference from current alternatives;
- reason to adopt now.

Output:

```text
Value proposition
```

---

# 5. Step 5 — Identify the Core Workflow

Map the minimum workflow that proves value.

Example structure:

```text
User starts with:
User does:
System helps by:
User finishes with:
```

Output:

```text
Core MVP workflow
```

---

# 6. Step 6 — List Candidate Features

List all proposed features without judging yet.

Group them by theme:

- onboarding;
- core workflow;
- collaboration;
- payments;
- reporting;
- admin;
- automation;
- AI;
- integrations.

Output:

```text
Candidate feature list
```

---

# 7. Step 7 — Select Must-Have Features

Keep only features required to complete the core workflow.

Must-have means:

- required for the user to reach value;
- required for the business to validate demand;
- required to operate the first release safely.

Output:

```text
MVP must-have feature list
```

---

# 8. Step 8 — Define Excluded Features

Document what is intentionally not included in the MVP.

Typical exclusions:

- advanced analytics;
- complex automation;
- multiple integrations;
- enterprise admin;
- deep customization;
- non-critical AI features;
- secondary personas.

Output:

```text
Excluded features and rationale
```

---

# 9. Step 9 — Define Success Metrics

Define how MVP success will be measured.

Possible metrics:

- activation rate;
- first workflow completion;
- retention;
- willingness to pay;
- number of qualified users;
- conversion to paid plan;
- time saved;
- qualitative feedback.

Output:

```text
MVP success metrics
```

---

# 10. Step 10 — Identify Risks and Assumptions

List what could make the MVP fail.

Examples:

- weak customer pain;
- unclear buyer;
- too much manual work;
- technical complexity;
- high acquisition cost;
- missing trust;
- poor onboarding;
- unclear pricing.

Output:

```text
Risk and assumption list
```

---

# 11. Step 11 — Produce First Roadmap

Sequence the MVP into phases.

Recommended structure:

```text
Phase 1 — Foundation
Phase 2 — Core workflow
Phase 3 — Launch readiness
Phase 4 — Feedback and iteration
```

Output:

```text
First MVP roadmap
```

---

# 12. Final Principle

> MVP definition is a reduction exercise: remove everything that does not prove the core value.
