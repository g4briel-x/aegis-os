# Aegis OS — Skill Authoring Guide

Version: 0.2  
Status: Skill Guide

---

# 1. Introduction

This guide explains how to write a high-quality Aegis OS Skill.

---

# 2. Authoring Philosophy

Aegis OS follows this principle:

> A Skill should encode how an expert thinks, decides, executes and validates.

---

# 3. Authoring Steps

Recommended process:

1. Define the expert role.
2. Define the scope.
3. Define required inputs.
4. Define expected outputs.
5. Write expertise rules.
6. Write workflows.
7. Write checklists.
8. Write reusable prompts.
9. Add examples.
10. Run quality review.

---

# 4. Writing Rules

A Skill should be:

- specific;
- operational;
- structured;
- domain-aware;
- validation-oriented.

Avoid:

- vague expertise;
- generic advice;
- unsupported claims;
- missing examples;
- unclear output expectations.

---

# 5. Final Principle

> The best Skills are written like expert operating manuals.
