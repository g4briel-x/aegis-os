Version: 0.2  
Status: Skill Governance

---

# 1. Introduction

This document defines how a Skill can become certified.

Certification means the Skill has passed stronger review and can be trusted for broader reuse.

---

# 2. Certification Requirements

A certified Skill should have:

- complete structure;
- valid metadata;
- reviewed expertise;
- tested workflows;
- strong examples;
- security and governance review when needed;
- clear owner.

---

# 3. Certification Flow

```text
Skill Candidate

   ↓

Structure Review

   ↓

Domain Review

   ↓

Execution Review

   ↓

Quality Gate

   ↓

Certification Decision
```

---

# 4. Final Principle

> Certification is a trust signal based on evidence.
