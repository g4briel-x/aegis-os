# Aegis OS — Skill Directory Structure

Version: 0.2  
Status: Skill Standard

---

# 1. Introduction

This document defines the standard folder structure for Skills v2.

---

# 2. Recommended Structure

```text
skills/
  engineering/
    senior-developer/
      README.md
      SKILL.md
      metadata.yaml
      expertise.md
      workflows.md
      checklists.md
      prompts.md
      examples/
        examples.md
```

---

# 3. Domain Folders

Recommended domains:

```text
engineering/
product/
design/
infrastructure/
security/
data/
management/
business/
operations/
```

---

# 4. Naming Rules

Skill folder names should:

- use lowercase;
- use hyphens;
- describe the role or capability clearly;
- avoid vague names.

Good examples:

```text
senior-developer
software-architect
product-manager-saas
security-engineer
technical-project-manager
```

---

# 5. Final Principle

> A predictable Skill structure makes the ecosystem easier to navigate, validate and extend.
