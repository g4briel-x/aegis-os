# Aegis OS — Skill Quality Gate

Version: 0.2  
Status: Skill Standard

---

# 1. Introduction

This document defines the quality gate for accepting Skills into Aegis OS.

---

# 2. Quality Dimensions

Each Skill should be checked for:

- structural completeness;
- domain accuracy;
- workflow clarity;
- prompt usefulness;
- example quality;
- output validation;
- maintainability.

---

# 3. Review Passes

Recommended review passes:

```text
Pass 1 — Structure Review
Pass 2 — Domain Review
Pass 3 — Execution Review
Pass 4 — Quality and Consistency Review
```

---

# 4. Acceptance Criteria

A Skill is acceptable when:

- required files exist;
- metadata is valid;
- workflows are actionable;
- checklists are useful;
- examples demonstrate real usage;
- prompts are reusable.

---

# 5. Final Principle

> A Skill should not enter the ecosystem until it can be trusted to guide real work.
