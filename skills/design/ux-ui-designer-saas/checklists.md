# UX/UI Designer SaaS — Checklists

Version: 0.2.0  
Status: Premium Draft

---

# 1. UX Discovery Checklist

```text
[ ] Target user identified
[ ] User goal defined
[ ] Business goal defined
[ ] Main use case defined
[ ] Current friction identified
[ ] Success state defined
[ ] Assumptions documented
```

---

# 2. User Flow Checklist

```text
[ ] Entry point defined
[ ] Primary action clear
[ ] Steps are ordered logically
[ ] Unnecessary steps removed
[ ] Error states considered
[ ] Empty states considered
[ ] Success state defined
[ ] Flow supports user intent
```

---

# 3. Screen Design Checklist

```text
[ ] Screen purpose clear
[ ] Primary action visible
[ ] Content hierarchy strong
[ ] Navigation clear
[ ] Forms are simple
[ ] Feedback states defined
[ ] Responsive behavior considered
[ ] Accessibility considered
```

---

# 4. SaaS Dashboard Checklist

```text
[ ] Dashboard user role defined
[ ] Key decisions supported
[ ] Metrics prioritized
[ ] Filters are useful
[ ] Tables or cards are readable
[ ] Empty states defined
[ ] Alerts and actions are clear
[ ] Layout avoids overload
```

---

# 5. Accessibility Checklist

```text
[ ] Contrast reviewed
[ ] Typography readable
[ ] Labels clear
[ ] Error messages helpful
[ ] Focus states considered
[ ] Keyboard flow considered
[ ] Icons have text support where needed
[ ] Color is not the only signal
```

---

# 6. 4-Pass Validation Checklist

```text
[ ] Pass 1 completed — user goal and journey
[ ] Pass 2 completed — information architecture and usability
[ ] Pass 3 completed — visual hierarchy, accessibility and responsiveness
[ ] Pass 4 completed — product clarity and implementation readiness
[ ] Weaknesses corrected or documented
```

---

# 7. Final Principle

> UX/UI checklists prevent attractive screens from hiding usability problems.
