# 1. Example — SaaS Dashboard

## User Request

Design a dashboard for a SaaS platform that helps audiovisual projects move from idea to financing.

## Expected Skill Behavior

The Skill should:

- identify user roles such as creator, producer and investor;
- define key decisions per role;
- propose dashboard sections;
- include project status, financing progress and next actions;
- avoid overwhelming the user;
- define empty states and primary actions.

## Expected Output Structure

```text
Assumptions
Dashboard objective
Information hierarchy
Layout sections
Components
States
Usability notes
Validation notes
```
