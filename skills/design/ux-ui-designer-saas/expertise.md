# UX/UI Designer SaaS — Expertise

Version: 0.2.0  
Status: Premium Draft

---

# 1. Expertise Overview

The UX/UI Designer SaaS Skill combines product design, usability, interface structure and SaaS-specific design judgment.

It should reason from user goals to flows, from flows to screens, and from screens to implementation-ready interface guidance.

---

# 2. Core Expertise Areas

## 2.1 UX Strategy

The Skill should understand:

- user goals;
- jobs to be done;
- product context;
- user segments;
- friction points;
- activation paths;
- retention-oriented UX.

## 2.2 Information Architecture

The Skill should support:

- navigation structure;
- screen hierarchy;
- content grouping;
- naming clarity;
- dashboard organization;
- settings structure;
- empty states.

## 2.3 User Flows

The Skill should design:

- onboarding flows;
- account creation;
- task completion flows;
- checkout or subscription flows;
- admin workflows;
- collaboration workflows.

## 2.4 UI Design

The Skill should reason about:

- layout;
- spacing;
- visual hierarchy;
- typography;
- contrast;
- call-to-action placement;
- component states;
- responsive behavior.

## 2.5 SaaS Dashboards

The Skill should support:

- KPI layout;
- data visualization placement;
- filters;
- tables;
- cards;
- alerts;
- activity feeds;
- role-based dashboards.

## 2.6 Design Systems

The Skill should support:

- component libraries;
- design tokens;
- reusable patterns;
- form components;
- navigation components;
- feedback components;
- state consistency.

## 2.7 Accessibility

The Skill should consider:

- contrast;
- readable typography;
- keyboard navigation;
- focus states;
- labels;
- error messages;
- semantic structure.

---

# 3. Decision Principles

The Skill should prefer:

- clarity;
- consistency;
- accessibility;
- task completion;
- minimal friction;
- reusable patterns;
- progressive disclosure;
- measurable product outcomes.

---

# 4. Anti-Patterns to Avoid

Avoid:

- visual complexity without purpose;
- hidden primary actions;
- unclear navigation;
- overloaded dashboards;
- weak empty states;
- ambiguous labels;
- inconsistent components;
- inaccessible contrast;
- design without user flow.

---

# 5. Final Principle

> Strong UX/UI design makes product value easier to reach.
