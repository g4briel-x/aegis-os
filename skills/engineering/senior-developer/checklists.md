# Senior Developer — Checklists

Version: 0.2.0  
Status: Premium Draft

---

# 1. General Delivery Checklist

```text
[ ] Requirement understood
[ ] Target stack identified
[ ] Assumptions stated
[ ] Solution is appropriate for the problem
[ ] Output is structured
[ ] Risks are mentioned when relevant
[ ] Next steps are clear
```

---

# 2. Code Quality Checklist

```text
[ ] Code is readable
[ ] Names are clear
[ ] Responsibilities are separated
[ ] Error handling is present where needed
[ ] No obvious duplication
[ ] No unnecessary complexity
[ ] Security concerns reviewed
[ ] Edge cases considered
```

---

# 3. 4-Pass Verification Checklist

```text
[ ] Pass 1 completed — syntax and obvious errors
[ ] Pass 2 completed — logic and edge cases
[ ] Pass 3 completed — security and maintainability
[ ] Pass 4 completed — final readiness
[ ] Detected errors corrected
```

---

# 4. Database Checklist

```text
[ ] Entities identified
[ ] Relationships defined
[ ] Primary keys defined
[ ] Foreign keys considered
[ ] Constraints defined
[ ] Indexes considered
[ ] Query performance reviewed
[ ] Data integrity protected
```

---

# 5. API Checklist

```text
[ ] Endpoint purpose clear
[ ] Request format defined
[ ] Response format defined
[ ] Validation included
[ ] Error responses defined
[ ] Authentication considered
[ ] Security reviewed
```

---

# 6. Debugging Checklist

```text
[ ] Error message reviewed
[ ] Context understood
[ ] Likely root cause identified
[ ] Fix proposed
[ ] Explanation provided
[ ] Prevention advice included
```

---

# 7. Final Principle

> Checklists protect delivery quality by making expert review explicit.
