# Software Architect — Checklists

Version: 0.2.0  
Status: Premium Draft

---

# 1. Architecture Discovery Checklist

```text
[ ] Business goal understood
[ ] Primary users identified
[ ] Core workflows identified
[ ] Functional requirements listed
[ ] Non-functional requirements listed
[ ] Constraints documented
[ ] Assumptions stated
```

---

# 2. System Architecture Checklist

```text
[ ] System scope is clear
[ ] Main components identified
[ ] Component responsibilities defined
[ ] Data ownership defined
[ ] Communication flows documented
[ ] External dependencies identified
[ ] Security boundaries defined
[ ] Deployment model considered
```

---

# 3. SaaS Architecture Checklist

```text
[ ] Tenant model defined
[ ] User roles defined
[ ] Subscription or billing boundaries considered
[ ] Data isolation considered
[ ] Admin features identified
[ ] Audit and logging considered
[ ] Scaling path considered
[ ] Support and operations considered
```

---

# 4. API Architecture Checklist

```text
[ ] API boundaries clear
[ ] Request and response models defined
[ ] Authentication considered
[ ] Authorization considered
[ ] Error conventions defined
[ ] Versioning approach considered
[ ] Rate limiting considered where relevant
```

---

# 5. Architecture Review Checklist

```text
[ ] Requirements alignment reviewed
[ ] Simplicity reviewed
[ ] Coupling reviewed
[ ] Scalability reviewed
[ ] Security reviewed
[ ] Maintainability reviewed
[ ] Operational readiness reviewed
[ ] Risks documented
```

---

# 6. 4-Pass Validation Checklist

```text
[ ] Pass 1 completed — requirements and scope
[ ] Pass 2 completed — structure and dependencies
[ ] Pass 3 completed — security, scalability and maintainability
[ ] Pass 4 completed — decision clarity and implementation readiness
[ ] Weaknesses corrected or documented
```

---

# 7. Final Principle

> Architecture checklists prevent elegant diagrams from hiding weak decisions.
