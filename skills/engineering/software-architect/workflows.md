# Software Architect — Workflows

Version: 0.2.0  
Status: Premium Draft

---

# 1. Architecture Discovery Workflow

Use this workflow when the user has an idea or product need.

```text
1. Understand the product goal
2. Identify users and core workflows
3. Extract functional requirements
4. Extract non-functional requirements
5. Identify constraints
6. Define architecture options
7. Compare trade-offs
8. Recommend a path
```

---

# 2. System Design Workflow

Use this workflow for new system architecture.

```text
1. Define system scope
2. Identify main components
3. Define data entities
4. Define communication flows
5. Define external integrations
6. Define security boundaries
7. Define deployment model
8. Validate risks and trade-offs
```

---

# 3. Architecture Review Workflow

Use this workflow when reviewing an existing architecture.

```text
1. Understand current architecture
2. Identify intended business goals
3. Review component boundaries
4. Review data ownership
5. Review security and access control
6. Review scalability and operations
7. Identify risks
8. Recommend improvements
```

---

# 4. Architecture Decision Workflow

Use this workflow when comparing options.

```text
1. Define the decision to make
2. Define decision criteria
3. List viable options
4. Compare benefits and costs
5. Identify risks
6. Recommend one option
7. Document consequences
```

---

# 5. SaaS Architecture Workflow

Use this workflow when designing SaaS platforms.

```text
1. Define target users and roles
2. Define tenant model
3. Define core modules
4. Define billing or subscription boundaries
5. Define data isolation strategy
6. Define admin and operational features
7. Define security and compliance needs
8. Produce phased implementation roadmap
```

---

# 6. 4-Pass Architecture Validation Workflow

Every architecture output must be reviewed using this workflow:

```text
Pass 1 — Requirement and scope alignment
Pass 2 — Structural coherence and dependency review
Pass 3 — Security, scalability and maintainability review
Pass 4 — Decision clarity and implementation readiness review
```

---

# 7. Final Principle

> Architecture work is complete only when the design is clear enough to guide implementation.
