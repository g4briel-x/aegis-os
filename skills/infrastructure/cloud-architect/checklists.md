# Cloud Architect — Checklists

Version: 0.2.0  
Status: Premium Draft

---

# 1. Cloud Architecture Checklist

```text
[ ] Workload requirements understood
[ ] Cloud provider identified or assumptions stated
[ ] Environments defined
[ ] Compute model selected
[ ] Database and storage selected
[ ] Networking boundaries defined
[ ] Security model considered
[ ] Observability considered
[ ] Cost risks reviewed
```

---

# 2. Security Checklist

```text
[ ] IAM roles reviewed
[ ] Least privilege considered
[ ] Public exposure reviewed
[ ] Secrets handling defined
[ ] Encryption considered
[ ] Audit logging considered
[ ] Sensitive data boundaries defined
```

---

# 3. Reliability Checklist

```text
[ ] Critical services identified
[ ] Health checks defined
[ ] Backup strategy defined
[ ] Restore process considered
[ ] Failover considered where needed
[ ] Recovery objectives defined
[ ] Incident response link considered
```

---

# 4. Cost Checklist

```text
[ ] Major cost drivers identified
[ ] Compute sizing considered
[ ] Storage growth considered
[ ] Network cost considered
[ ] Idle resource risk reviewed
[ ] Cost monitoring recommended
[ ] Scaling cost considered
```

---

# 5. Observability Checklist

```text
[ ] Logs defined
[ ] Metrics defined
[ ] Alerts defined
[ ] Dashboards considered
[ ] Error rate monitored
[ ] Latency monitored where relevant
[ ] Capacity signals considered
```

---

# 6. 4-Pass Validation Checklist

```text
[ ] Pass 1 completed — requirements and workload
[ ] Pass 2 completed — architecture and networking
[ ] Pass 3 completed — security and identity
[ ] Pass 4 completed — cost, reliability and observability
[ ] Weaknesses corrected or documented
```

---

# 7. Final Principle

> Cloud checklists protect the system from hidden infrastructure, security and cost assumptions.
