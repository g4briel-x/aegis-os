# 1. Example — SaaS Cloud Architecture

## User Request

Design a cloud architecture for a SaaS platform that helps audiovisual projects move from idea to financing.

## Expected Skill Behavior

The Skill should:

- define environments;
- propose application, database and storage layers;
- define tenant and access boundaries;
- include monitoring and backup strategy;
- consider early-stage cost control;
- avoid unnecessary complexity.

## Expected Output Structure

```text
Assumptions
Cloud architecture
Services
Security model
Deployment topology
Observability
Cost notes
Risks
Validation notes
```

# 2. Example — Cloud Cost Review

## User Request

Our cloud bill is increasing. Review possible causes and optimization actions.

## Expected Skill Behavior

The Skill should:

- identify cost drivers;
- ask for provider and usage details if missing;
- recommend quick checks;
- distinguish safe savings from risky cuts;
- propose monitoring.

# 3. Example — Migration Plan

## User Request

Move a small Node.js and PostgreSQL application from a VPS to cloud infrastructure.

## Expected Skill Behavior

The Skill should:

- define target architecture;
- plan database backup and restore;
- define cutover;
- define rollback;
- validate DNS and monitoring.

---

# 4. Example — Security Review

## User Request

Review whether our cloud storage bucket should be public.

## Expected Skill Behavior

The Skill should:

- challenge public exposure;
- define access alternatives;
- recommend least privilege;
- mention audit and logging;
- document risk.

# 5. Example — Disaster Recovery

## User Request

Create a disaster recovery plan for a SaaS backend.

## Expected Skill Behavior

The Skill should:

- identify critical components;
- define backups;
- define restore steps;
- define RPO and RTO assumptions;
- define verification tests.

> Examples prove that the Skill can design cloud systems that are practical, secure and recoverable.
