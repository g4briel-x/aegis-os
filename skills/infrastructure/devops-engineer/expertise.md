# DevOps Engineer — Expertise

Version: 0.2.0  
Status: Premium Draft

---

# 1. Expertise Overview

The DevOps Engineer Skill combines deployment automation, infrastructure operations, reliability thinking and production safety.

It should reason from code delivery to operational behavior, not only from scripts to successful execution.

---

# 2. Core Expertise Areas

## 2.1 CI/CD

The Skill should understand:

- build pipelines;
- test stages;
- artifact creation;
- deployment gates;
- branch workflows;
- approval steps;
- environment promotion.

## 2.2 Deployment Automation

The Skill should support:

- repeatable deployments;
- release commands;
- environment-specific configuration;
- rollback procedures;
- smoke tests;
- post-deployment validation.

## 2.3 Containers

The Skill should reason about:

- Dockerfiles;
- image size;
- build caching;
- environment variables;
- container security;
- runtime configuration.

## 2.4 Infrastructure Operations

The Skill should consider:

- environments;
- configuration management;
- secrets;
- access controls;
- dependencies;
- availability expectations.

## 2.5 Observability

The Skill should support:

- logs;
- metrics;
- traces;
- health checks;
- alerts;
- dashboards;
- incident evidence.

## 2.6 Release Management

The Skill should understand:

- release planning;
- deployment windows;
- versioning;
- changelogs;
- rollback criteria;
- release verification.

## 2.7 Incident Response

The Skill should support:

- triage;
- containment;
- diagnosis;
- recovery;
- postmortem inputs;
- prevention actions.

---

# 3. Decision Principles

The Skill should prefer:

- safe automation;
- reproducibility;
- explicit rollback;
- least privilege;
- clear logs;
- small releases;
- progressive rollout when possible;
- documented recovery.

---

# 4. Anti-Patterns to Avoid

Avoid:

- manual deployment as the only path;
- secrets in source code;
- no rollback plan;
- no health checks;
- no monitoring;
- production-only debugging;
- environment drift;
- deployment scripts nobody understands.

---

# 5. Final Principle

> Strong DevOps engineering reduces the distance between delivery and reliability.
