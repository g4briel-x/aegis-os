# Technical Project Manager — Checklists

Version: 0.2.0  
Status: Premium Draft

---

# 1. Project Plan Checklist

```text
[ ] Objective defined
[ ] Scope defined
[ ] Non-scope defined
[ ] Stakeholders identified
[ ] Deliverables listed
[ ] Milestones defined
[ ] Owners assigned
[ ] Dependencies identified
[ ] Risks documented
```

---

# 2. Milestone Checklist

```text
[ ] Milestone outcome clear
[ ] Deliverables defined
[ ] Completion criteria defined
[ ] Owner identified
[ ] Dependencies identified
[ ] Risks identified
[ ] Review point defined
[ ] Status measurable
```

---

# 3. Status Report Checklist

```text
[ ] Overall status clear
[ ] Completed work summarized
[ ] Current work listed
[ ] Blockers visible
[ ] Risks visible
[ ] Decisions needed listed
[ ] Next actions assigned
[ ] Audience-appropriate language used
```

---

# 4. Risk Register Checklist

```text
[ ] Risk described clearly
[ ] Impact defined
[ ] Likelihood defined
[ ] Owner assigned
[ ] Mitigation defined
[ ] Escalation path defined
[ ] Status tracked
```

---

# 5. Release Readiness Checklist

```text
[ ] Release scope confirmed
[ ] Acceptance criteria met
[ ] QA status confirmed
[ ] Deployment plan ready
[ ] Rollback plan ready
[ ] Communication prepared
[ ] Monitoring plan defined
[ ] Post-release owner assigned
```

---

# 6. 4-Pass Validation Checklist

```text
[ ] Pass 1 completed — scope and stakeholders
[ ] Pass 2 completed — timeline and dependencies
[ ] Pass 3 completed — risks and decisions
[ ] Pass 4 completed — communication and ownership
[ ] Weaknesses corrected or documented
```

---

# 7. Final Principle

> Checklists keep technical delivery honest, visible and accountable.
