# 1. Example — SaaS MVP Delivery Plan

## User Request

Create a delivery plan for the MVP of a SaaS platform that helps audiovisual projects move from idea to financing.

## Expected Skill Behavior

The Skill should:

- define scope and non-scope;
- identify product, design, engineering and security workstreams;
- create milestones;
- identify dependencies;
- define risks;
- assign owners as placeholders if unknown;
- define communication cadence.

## Expected Output Structure

```text
Assumptions
Objective
Scope
Milestones
Workstreams
Dependencies
Risks
Owners
Communication plan
Validation notes
```

# 2. Example — Project Status Report

## User Request

Create a weekly status report for a delayed backend API project.

## Expected Skill Behavior

The Skill should:

- state overall status;
- identify delay reason;
- list blockers;
- define mitigation actions;
- identify decisions needed;
- keep stakeholder tone clear.

---

# 3. Example — Risk Register

## User Request

Create a risk register for a cloud migration.

## Expected Skill Behavior

The Skill should include:

- data migration risk;
- downtime risk;
- DNS cutover risk;
- access control risk;
- rollback risk;
- mitigation and owners.

# 4. Example — Sprint Plan

## User Request

Plan a sprint for user authentication, role management and dashboard setup.

## Expected Skill Behavior

The Skill should:

- define sprint goal;
- break down work items;
- identify dependencies;
- include acceptance criteria;
- flag security review needs.

# 5. Example — Release Plan

## User Request

Prepare a release plan for a new billing feature.

## Expected Skill Behavior

The Skill should:

- confirm release scope;
- include QA checks;
- include payment provider validation;
- define rollback plan;
- define communication plan;
- include post-release monitoring.
