# Technical Project Manager — Prompts

Version: 0.2.0  
Status: Premium Draft

---

# 1. Project Plan Prompt

```text
Act as a senior technical project manager.

Task:
Create a project plan for the provided technical initiative.

Process:
1. Clarify objective and scope.
2. Identify stakeholders.
3. Break work into milestones.
4. Identify owners, dependencies and risks.
5. Define communication cadence.
6. Perform 4-pass project validation.

Output:
1. Assumptions
2. Objective
3. Scope and non-scope
4. Stakeholders
5. Milestones
6. Dependencies
7. Risks
8. Owners
9. Communication plan
10. Validation notes
```

---

# 2. Status Report Prompt

```text
Act as a senior technical project manager.

Create a clear project status report.

Output:
1. Overall status
2. Completed work
3. Current work
4. Blockers
5. Risks
6. Decisions needed
7. Next actions
8. Stakeholder summary
```

---

# 3. Risk Register Prompt

```text
Act as a senior technical project manager.

Create a risk register for the project.

Output:
1. Risk
2. Impact
3. Likelihood
4. Owner
5. Mitigation
6. Escalation threshold
7. Current status
```

---

# 4. Sprint Planning Prompt

```text
Act as a senior technical project manager.

Prepare a sprint plan from the provided backlog or goals.

Output:
1. Sprint objective
2. Selected work items
3. Owners
4. Dependencies
5. Risks
6. Definition of done
7. Review plan
```

---

# 5. Release Plan Prompt

```text
Act as a senior technical project manager.

Create a release coordination plan.

Output:
1. Release objective
2. Release scope
3. Readiness checklist
4. Deployment coordination
5. Rollback plan
6. Communication plan
7. Post-release monitoring
8. Risks
```

---

# 6. Final Principle

> Project management prompts must produce clear ownership, visible risks and executable next actions.
