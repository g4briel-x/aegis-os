# Business Analyst — Checklists

Version: 0.2.0  
Status: Premium Draft

---

# 1. Requirement Checklist

```text
[ ] Business objective defined
[ ] Stakeholders identified
[ ] Requirement is clear
[ ] Requirement is testable
[ ] Business value is stated
[ ] Assumptions are documented
[ ] Dependencies are identified
[ ] Acceptance criteria are defined
```

---

# 2. Stakeholder Checklist

```text
[ ] Primary users identified
[ ] Decision makers identified
[ ] Operators identified where relevant
[ ] Support impact considered
[ ] Conflicting expectations identified
[ ] Approval path documented
```

---

# 3. User Story Checklist

```text
[ ] User role is clear
[ ] User goal is clear
[ ] Business value is clear
[ ] Acceptance criteria are specific
[ ] Edge cases are considered
[ ] Dependencies are noted
[ ] Story is small enough to execute
```

---

# 4. Process Checklist

```text
[ ] Current state documented
[ ] Future state documented
[ ] Actors identified
[ ] Handoffs identified
[ ] Exceptions considered
[ ] Bottlenecks identified
[ ] Automation opportunities identified
[ ] Change impact documented
```

---

# 5. Data Requirement Checklist

```text
[ ] Data entities identified
[ ] Required fields listed
[ ] Data sources identified
[ ] Data ownership considered
[ ] Reporting needs documented
[ ] Data quality rules considered
[ ] Audit needs considered
```

---

# 6. 4-Pass Validation Checklist

```text
[ ] Pass 1 completed — business objective and stakeholders
[ ] Pass 2 completed — requirement clarity and completeness
[ ] Pass 3 completed — process, data and dependencies
[ ] Pass 4 completed — acceptance, risks and implementation readiness
[ ] Weaknesses corrected or documented
```

---

# 7. Final Principle

> Checklists make requirements safer by forcing clarity before implementation.
