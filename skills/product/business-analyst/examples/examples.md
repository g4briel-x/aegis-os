# 1. Example — SaaS Requirements

## User Request

Define the requirements for a SaaS platform that helps audiovisual projects move from idea to financing.

## Expected Skill Behavior

The Skill should:

- identify stakeholders such as creators, producers and investors;
- define business objectives;
- identify functional requirements;
- define non-functional requirements;
- define acceptance criteria;
- list risks and open questions.

## Expected Output Structure

```text
Context
Stakeholders
Functional requirements
Non-functional requirements
Acceptance criteria
Risks
Open questions
Validation notes
```

# 2. Example — User Stories

## User Request

Create user stories for a client portal where customers can view invoices and project status.

## Expected Skill Behavior

The Skill should:

- identify customer role;
- define user goals;
- write testable user stories;
- include acceptance criteria;
- include edge cases.

# 3. Example — Process Mapping

## User Request

Analyze our current manual approval process for project funding requests.

## Expected Skill Behavior

The Skill should:

- map current workflow;
- identify actors;
- identify bottlenecks;
- propose future-state flow;
- define system requirements.

# 4. Example — Gap Analysis

## User Request

Compare our current Airtable-based workflow with a future SaaS platform.

## Expected Skill Behavior

The Skill should:

- document current state;
- define target state;
- identify data, process and technical gaps;
- recommend priority actions.

# 5. Example — Acceptance Criteria

## User Request

Write acceptance criteria for a login and role-based access feature.

## Expected Skill Behavior

The Skill should include:

- successful login;
- failed login;
- role-based access;
- session behavior;
- security expectations;
- error messages.
