Version: 0.2.0  
Status: Premium Draft

# 1. Requirements Analysis Prompt

```text
Act as a senior business analyst.

Task:
Analyze the provided business need and convert it into clear requirements.

Process:
1. Clarify business objective.
2. Identify stakeholders.
3. Separate assumptions from facts.
4. Define functional requirements.
5. Define non-functional requirements.
6. Define acceptance criteria.
7. Perform 4-pass business analysis validation.

Output:
1. Context
2. Stakeholders
3. Assumptions
4. Functional requirements
5. Non-functional requirements
6. Acceptance criteria
7. Risks and dependencies
8. Open questions
9. Validation notes
```

---

# 2. User Story Prompt

```text
Act as a senior business analyst.

Convert the requirement into user stories.

Output:
1. User stories
2. Acceptance criteria
3. Edge cases
4. Dependencies
5. Notes for product/design/engineering
```

# 3. BRD Prompt

```text

Output:
```

# 4. Process Mapping Prompt

```text
Act as a senior business analyst.

Map and analyze the provided business process.

Output:
1. Process objective
2. Current-state workflow
3. Actors and responsibilities
4. Pain points
5. Future-state workflow
6. System requirements
7. Risks and dependencies
8. Improvement opportunities
```

---

# 5. Gap Analysis Prompt

```text
Act as a senior business analyst.

Perform a gap analysis between the current state and the target state.

Output:
1. Current state
2. Target state
3. Capability gaps
4. Process gaps
5. Data gaps
6. Technical gaps
7. Prioritized recommendations
8. Next actions
```

# 6. Final Principle

> Business analysis prompts must turn vague intent into precise, testable and useful requirements.
