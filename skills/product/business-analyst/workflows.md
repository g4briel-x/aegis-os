# Business Analyst — Workflows

Version: 0.2.0  
Status: Premium Draft

---

# 1. Requirement Discovery Workflow

Use this workflow when clarifying a business need.

```text
1. Identify the business objective
2. Identify stakeholders
3. Capture the current problem
4. Separate facts from assumptions
5. Identify user needs
6. Define candidate requirements
7. Validate scope
8. Document open questions
```

---

# 2. BRD Workflow

Use this workflow when creating a Business Requirements Document.

```text
1. Define background and context
2. Define business problem
3. Define objectives
4. Define stakeholders
5. Define scope and non-scope
6. Define requirements
7. Define risks and dependencies
8. Define acceptance and success criteria
```

---

# 3. User Story Workflow

Use this workflow when converting requirements into user stories.

```text
1. Identify user role
2. Identify user goal
3. Define business value
4. Write user story
5. Define acceptance criteria
6. Identify edge cases
7. Identify dependencies
8. Validate testability
```

---

# 4. Process Analysis Workflow

Use this workflow when analyzing a workflow or business process.

```text
1. Define process objective
2. Map current steps
3. Identify actors and handoffs
4. Identify bottlenecks
5. Identify exceptions
6. Define future-state flow
7. Identify system requirements
8. Document risks and changes
```

---

# 5. Gap Analysis Workflow

Use this workflow when comparing current and desired states.

```text
1. Define current state
2. Define desired state
3. Identify capability gaps
4. Identify process gaps
5. Identify data gaps
6. Identify technical gaps
7. Prioritize gaps
8. Recommend next actions
```

---

# 6. 4-Pass Business Analysis Validation Workflow

Every major business analysis output must be reviewed using this workflow:

```text
Pass 1 — Business objective and stakeholder alignment
Pass 2 — Requirement clarity and completeness review
Pass 3 — Process, data and dependency review
Pass 4 — Acceptance, risk and implementation-readiness review
```

---

# 7. Final Principle

> Business analysis work is complete only when the next product, design or engineering action is clear.
