# Product Manager SaaS — Checklists

Version: 0.2.0  
Status: Premium Draft

---

# 1. Product Idea Checklist

```text
[ ] Target customer defined
[ ] Problem clearly stated
[ ] Current alternatives identified
[ ] Value proposition drafted
[ ] Assumptions listed
[ ] Validation method defined
[ ] Business objective identified
```

---

# 2. MVP Checklist

```text
[ ] Core user problem defined
[ ] Primary persona defined
[ ] Essential use case selected
[ ] Non-essential features removed
[ ] Success metrics defined
[ ] Launch risks documented
[ ] Dependencies identified
[ ] First release is realistic
```

---

# 3. PRD Checklist

```text
[ ] Context included
[ ] Problem statement clear
[ ] Goals defined
[ ] Non-goals defined
[ ] User stories included
[ ] Acceptance criteria included
[ ] Success metrics defined
[ ] Risks and open questions documented
```

---

# 4. Roadmap Checklist

```text
[ ] Strategic objective defined
[ ] Initiatives grouped by theme
[ ] Priorities justified
[ ] Dependencies identified
[ ] Delivery sequence is realistic
[ ] Metrics attached to major initiatives
[ ] Stakeholder communication is clear
```

---

# 5. SaaS Metrics Checklist

```text
[ ] Activation metric defined
[ ] Retention metric defined
[ ] Conversion metric defined
[ ] Revenue metric defined
[ ] Churn considered
[ ] Expansion opportunity considered
[ ] Metric owners identified where relevant
```

---

# 6. 4-Pass Validation Checklist

```text
[ ] Pass 1 completed — problem, user and market alignment
[ ] Pass 2 completed — business model and value proposition
[ ] Pass 3 completed — scope, roadmap and feasibility
[ ] Pass 4 completed — metrics, risks and decision readiness
[ ] Weaknesses corrected or documented
```

---

# 7. Final Principle

> Product checklists protect against attractive ideas that are not yet validated, prioritized or executable.
