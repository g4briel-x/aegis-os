# Product Manager SaaS — Workflows

Version: 0.2.0  
Status: Premium Draft

---

# 1. Product Discovery Workflow

Use this workflow when validating a product idea.

```text
1. Define the target customer
2. Define the problem hypothesis
3. Identify assumptions
4. Plan discovery questions
5. Collect evidence
6. Synthesize pain points
7. Evaluate willingness to pay
8. Recommend next decision
```

---

# 2. MVP Definition Workflow

Use this workflow when defining a first version.

```text
1. Clarify user problem
2. Define target segment
3. Identify core job to be done
4. List candidate features
5. Remove non-essential features
6. Define success metrics
7. Define launch scope
8. Document risks and dependencies
```

---

# 3. PRD Workflow

Use this workflow when creating a product requirements document.

```text
1. Define context
2. Define problem
3. Define goals and non-goals
4. Define users and use cases
5. Define functional requirements
6. Define non-functional requirements
7. Define success metrics
8. Define risks, dependencies and open questions
```

---

# 4. Feature Prioritization Workflow

Use this workflow when ranking product work.

```text
1. List candidate features
2. Define scoring criteria
3. Estimate impact
4. Estimate effort
5. Estimate confidence
6. Identify dependencies
7. Rank features
8. Recommend roadmap sequence
```

---

# 5. Roadmap Workflow

Use this workflow when planning product evolution.

```text
1. Define strategic objective
2. Define planning horizon
3. Group initiatives by theme
4. Sequence based on dependencies
5. Identify delivery risks
6. Define measurable outcomes
7. Communicate roadmap clearly
```

---

# 6. Pricing and Packaging Workflow

Use this workflow when defining SaaS monetization.

```text
1. Define customer segments
2. Identify value drivers
3. Define packaging logic
4. Define pricing tiers
5. Map features to willingness to pay
6. Identify expansion paths
7. Validate against market alternatives
```

---

# 7. 4-Pass Product Validation Workflow

Every major product output must be reviewed using this workflow:

```text
Pass 1 — Problem, user and market alignment
Pass 2 — Business model and value proposition review
Pass 3 — Scope, roadmap and execution feasibility review
Pass 4 — Metrics, risks and decision-readiness review
```

---

# 8. Final Principle

> Product workflow is complete only when the next decision is clear.
