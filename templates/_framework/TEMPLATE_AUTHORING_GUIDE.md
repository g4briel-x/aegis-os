# Template Authoring Guide

Version: 0.1.0  
Status: Foundation Draft

---

# 1. Purpose

This guide explains how to create a useful Aegis OS Template.

---

# 2. Authoring Process

Use this sequence:

```text
1. Identify repeatable output
2. Define target user and use case
3. Define required inputs
4. Define variables and placeholders
5. Define document sections
6. Add writing rules
7. Add quality checklist
8. Add example output
9. Link related skills, playbooks and patterns
10. Review for clarity and completeness
```

---

# 3. Writing Rules

A Template should:

- use clear section headings;
- define what belongs in each section;
- mark required and optional sections;
- include placeholders where useful;
- avoid vague instructions;
- include examples;
- support review and reuse.

---

# 4. Placeholder Style

Use clear placeholders:

```text
{{project_name}}
{{target_user}}
{{problem_statement}}
{{success_metric}}
{{owner}}
{{due_date}}
```

Avoid unclear placeholders:

```text
{{stuff}}
{{details}}
{{misc}}
```

---

# 5. Common Mistakes

Avoid:

```text
Too many empty sections
No variable list
No usage instructions
No quality checklist
Generic output
No example
No related assets
```

---

# 6. Final Principle

> Good Templates reduce blank-page friction and improve output quality.
