# Architecture Decision Record Template — Checklists

Version: 0.1.0  
Status: Draft

---

# 1. Completeness Checklist

```text
[ ] ADR title is clear
[ ] Decision owner is assigned
[ ] Context is documented
[ ] Problem statement is clear
[ ] Decision drivers are listed
[ ] Options are compared
[ ] Chosen option is explicit
[ ] Consequences are documented
[ ] Risks are documented
[ ] Follow-up actions are assigned
```

---

# 2. Decision Quality Checklist

```text
[ ] Decision solves the stated problem
[ ] Alternatives are realistic
[ ] Trade-offs are honest
[ ] Security impact is reviewed if relevant
[ ] Operations impact is reviewed if relevant
[ ] Data impact is reviewed if relevant
[ ] Future maintainers can understand the reasoning
```

---

# 3. Governance Checklist

```text
[ ] Status is current
[ ] Approval is recorded
[ ] Related documents are linked
[ ] Implementation work is linked
[ ] Review date is defined if needed
[ ] Superseded decisions are referenced if relevant
```

---

# 4. Final Principle

> ADR checklists should prevent important decisions from becoming undocumented assumptions.
