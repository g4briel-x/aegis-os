# Pricing Strategy Template — Checklists

Version: 0.1.0  
Status: Draft

---

# 1. Completeness Checklist

```text
[ ] Product or offer is defined
[ ] Target customer is defined
[ ] Customer value is described
[ ] Pricing objective is defined
[ ] Pricing model is selected
[ ] Value metric is defined
[ ] Plan structure exists
[ ] Entitlement matrix exists
[ ] Validation method is defined
[ ] Launch decision is recorded
```

---

# 2. Pricing Quality Checklist

```text
[ ] Price connects to value
[ ] Plans are easy to explain
[ ] Feature limits are clear
[ ] Upgrade path is logical
[ ] Downgrade rules are understood
[ ] Discounts are controlled
[ ] Paid pilot or trial rules are clear
```

---

# 3. Validation Checklist

```text
[ ] Willingness-to-pay questions are prepared
[ ] Target sample is defined
[ ] Success criteria are measurable
[ ] Objections are captured
[ ] Price sensitivity is reviewed
[ ] Customer feedback informs next pricing iteration
```

---

# 4. Implementation Checklist

```text
[ ] Billing provider is selected or deferred
[ ] Plan catalog is required or deferred
[ ] Entitlements are mapped
[ ] Payment failure behavior is considered
[ ] Invoice behavior is considered
[ ] Billing audit events are considered
```

---

# 5. Final Principle

> Pricing checklists should prevent vague monetization from becoming product complexity.
