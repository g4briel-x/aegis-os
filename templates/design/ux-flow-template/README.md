# 1. Purpose

The UX Flow Template provides a reusable structure for documenting how a user moves through a SaaS workflow from entry point to outcome.

It helps product, UX/UI, engineering and QA teams align on user goals, screens, states, actions, permissions, errors, empty states, edge cases, analytics and acceptance criteria before design or implementation.

---

# 2. When to Use

Use this Template when:

```text
a SaaS workflow must be designed
a new feature requires multiple screens
onboarding needs to be clarified
a PRD needs UX detail
engineering needs screen-by-screen behavior
QA needs user journey test cases
empty, loading and error states must be defined
```
