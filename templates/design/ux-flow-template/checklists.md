# UX Flow Template — Checklists

Version: 0.1.0  
Status: Draft

---

# 1. Completeness Checklist

```text
[ ] Flow name is defined
[ ] Primary user is defined
[ ] User goal is defined
[ ] Entry point is defined
[ ] Successful exit is defined
[ ] Main steps are mapped
[ ] Screens are listed
[ ] States are defined
[ ] Acceptance criteria are included
```

---

# 2. UX Quality Checklist

```text
[ ] Primary action is clear
[ ] User always knows next step
[ ] Empty state guides action
[ ] Error messages are actionable
[ ] Success state confirms outcome
[ ] Flow avoids unnecessary friction
[ ] Accessibility notes are included
```

---

# 3. Engineering Readiness Checklist

```text
[ ] System responses are defined
[ ] Data requirements are defined
[ ] Permissions are defined
[ ] Edge cases are included
[ ] Analytics events are listed
[ ] Side effects are documented
[ ] Acceptance criteria are testable
```

---

# 4. Security Checklist

```text
[ ] Protected actions require permission
[ ] Tenant boundary is defined if relevant
[ ] Sensitive data display is reviewed
[ ] Permission denied state exists
[ ] Audit events are defined if needed
```

---

# 5. Final Principle

> UX flow checklists should prove that the design can be implemented and tested without guesswork.
