# 1. Purpose

The Test Plan Template provides a reusable structure for planning how a SaaS feature, workflow, API, release or system change will be tested.

It helps engineering, QA, product, security and operations teams align on test scope, scenarios, risks, environments, data, automation, acceptance criteria and release confidence.

---

# 2. When to Use

Use this Template when:

```text
a new feature is being implemented
a PRD needs test coverage
an API contract must be validated
a production release is being prepared
a security-sensitive workflow changes
a bug fix needs regression coverage
a migration or integration needs verification
```
