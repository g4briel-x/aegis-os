# Test Plan Template — Checklists

Version: 0.1.0  
Status: Draft

---

# 1. Completeness Checklist

```text
[ ] Change name is defined
[ ] Test owner is assigned
[ ] Scope is defined
[ ] Non-scope is defined
[ ] References are linked
[ ] Test objectives are defined
[ ] Test environment is defined
[ ] Test data is defined
[ ] Exit criteria are defined
```

---

# 2. Coverage Checklist

```text
[ ] Happy path tests included
[ ] Validation tests included
[ ] Error tests included
[ ] Authorization tests included if relevant
[ ] Tenant isolation tests included if relevant
[ ] Regression areas listed
[ ] Edge cases included
```

---

# 3. Automation Checklist

```text
[ ] Automated tests are identified
[ ] Manual tests are identified
[ ] Automation gaps are documented
[ ] CI impact is reviewed
[ ] Test ownership is assigned
[ ] Flaky test risk is reviewed
```

---

# 4. Release Readiness Checklist

```text
[ ] Critical tests pass
[ ] High-priority defects are handled
[ ] Observability is verified if needed
[ ] Rollback or fallback is understood
[ ] Remaining risk is documented
[ ] Approvals are recorded
```

---

# 5. Final Principle

> Test Plan checklists should prove that release confidence is earned, not assumed.
