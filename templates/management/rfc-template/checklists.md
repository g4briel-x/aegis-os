# RFC Template — Checklists

Version: 0.1.0  
Status: Draft

---

# 1. Completeness Checklist

```text
[ ] RFC title is clear
[ ] Author is listed
[ ] Owner is assigned
[ ] Problem statement is clear
[ ] Goals are defined
[ ] Non-goals are defined
[ ] Proposal is specific
[ ] Options are compared
[ ] Recommended option is clear
[ ] Decision is recorded
```

---

# 2. Review Quality Checklist

```text
[ ] Product impact is reviewed
[ ] Engineering impact is reviewed
[ ] Security impact is reviewed
[ ] Operations impact is reviewed
[ ] Customer impact is reviewed
[ ] Risks are documented
[ ] Mitigations are documented
```

---

# 3. Decision Checklist

```text
[ ] Decision owner is clear
[ ] Reviewers are listed
[ ] Open questions are visible
[ ] Follow-up actions are assigned
[ ] Approval is recorded
[ ] Deferred items are tracked
```

---

# 4. Final Principle

> RFC checklists should make proposals reviewable and decisions traceable.
