# Postmortem Template — Checklists

Version: 0.1.0  
Status: Draft

---

# 1. Completeness Checklist

```text
[ ] Incident name is defined
[ ] Severity is documented
[ ] Owner is assigned
[ ] Executive summary is written
[ ] Customer impact is documented
[ ] Timeline is complete
[ ] Detection method is documented
[ ] Root cause is documented
[ ] Resolution is documented
[ ] Corrective actions are assigned
```

---

# 2. Timeline Checklist

```text
[ ] First symptom included
[ ] Detection time included
[ ] Incident declaration included
[ ] Customer report included if applicable
[ ] Mitigation time included
[ ] Resolution time included
[ ] Communication events included
[ ] Closure time included
```

---

# 3. Root Cause Checklist

```text
[ ] Direct cause is identified
[ ] Contributing factors are listed
[ ] Root cause is specific
[ ] Prevention gap is explained
[ ] Detection gap is explained if relevant
[ ] Response gap is explained if relevant
```

---

# 4. Corrective Action Checklist

```text
[ ] Each action has owner
[ ] Each action has due date
[ ] Each action has priority
[ ] Each action is specific
[ ] Actions cover prevention if needed
[ ] Actions cover detection if needed
[ ] Actions cover response if needed
```

---

# 5. Review Checklist

```text
[ ] Incident team reviewed
[ ] Operations reviewed
[ ] Engineering reviewed
[ ] Security reviewed if needed
[ ] Support reviewed if customer-visible
[ ] Follow-up review date is set
```

---

# 6. Final Principle

> Postmortem checklists should ensure learning becomes accountable action.
