# 1. Purpose

The PRD Template provides a reusable structure for writing clear Product Requirements Documents for SaaS features, MVPs and product workflows.

It helps product, design, engineering, security and operations teams align on the problem, target users, scope, requirements, user flows, success metrics, risks, dependencies and release readiness.

---

# 2. When to Use

Use this Template when:

```text
a new SaaS feature must be specified
an MVP needs product requirements
a workflow must be clarified before implementation
engineering needs acceptance criteria
design needs user flow context
stakeholders need a shared product decision document
```

---

# 3. Output

This Template produces:

```text
Product Requirements Document
```
