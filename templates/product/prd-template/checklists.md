# PRD Template — Checklists

Version: 0.1.0  
Status: Draft

---

# 1. Completeness Checklist

```text
[ ] Feature name is defined
[ ] Owner is assigned
[ ] Problem statement is clear
[ ] Goals are measurable
[ ] Non-goals are listed
[ ] Target users are defined
[ ] Use cases are included
[ ] Requirements are testable
[ ] Acceptance criteria are included
[ ] Success metrics are defined
```

---

# 2. Product Quality Checklist

```text
[ ] PRD solves a real user or business problem
[ ] Scope is focused
[ ] MVP/non-MVP boundary is clear
[ ] User journey is understandable
[ ] Dependencies are documented
[ ] Risks are documented
[ ] Open questions are visible
```

---

# 3. Engineering Readiness Checklist

```text
[ ] Functional requirements are implementable
[ ] Data requirements are defined
[ ] API requirements are defined if needed
[ ] Permissions are defined if needed
[ ] Error states are considered
[ ] Acceptance criteria are testable
[ ] Release plan is clear
```

---

# 4. Security and Operations Checklist

```text
[ ] Roles and permissions are reviewed
[ ] Audit events are defined if sensitive
[ ] Tenant isolation impact is reviewed
[ ] Feature flag need is reviewed
[ ] Rollback plan is defined if needed
[ ] Monitoring or analytics events are defined
```

---

# 5. Approval Checklist

```text
[ ] Product approved
[ ] Design approved
[ ] Engineering approved
[ ] Security approved if required
[ ] Operations approved if required
[ ] Stakeholders aligned
```

---

# 6. Final Principle

> A PRD is ready when teams can build from it without guessing the intent.
